Haiku (operating system)

Haiku, originally OpenBeOS, is a free and open-source operating system for personal computers. It is a community-driven continuation of BeOS and aims to be binary-compatible with it, but is largely a reimplementation with the exception of certain components like the Deskbar. The Haiku project began in 2001, supported by the nonprofit Haiku Inc., and the operating system remains in beta.

History and project

On 17 August 2001 Palm, Inc. announced the purchase of Be, Inc., marking the end of BeOS development. The day after, Michael Phipps started the OpenBeOS project[1] to support the BeOS user community by creating an open-source, backward-compatible replacement for BeOS.[2] Palm refused to license the BeOS code to a third-party,[3] meaning that OpenBeOS had to be reverse-engineered.[4] In February 2002, the project released its first test build.[5] In 2003, Phipps founded the non-profit organization Haiku, Inc. in Rochester, New York, United States, to financially support development.[6] In 2004, the project held its first North American developers' conference, WalterCon ;[7] it was also announced on this day, June 19, that OpenBeOS was renamed to Haiku[2] to avoid infringing on Palm's trademarks.[8] The BeUnited.org nonprofit organization, which promoted open standards for BeOS-compatible operating system projects,[9] announced that Haiku would be its "reference platform".[10] In February 2007, the project held a Tech Talk at Googleplex, attended by ex-Be engineers as well as Jean-Louis GassÃ©e who voiced his support for the project.[11][7] There is also an annual conference, BeGeistert, held in Germany since 1998 when BeOS was active.[12][13]

Development

Apart from the graphical user interface (Tracker and Deskbar, which were open sourced with BeOS 5), Haiku is original software.[14] The modular design[15] of BeOS allowed individual components of Haiku to initially be developed in teams in relative isolation, in many cases developing them as replacements for the BeOS components prior to the completion of other parts of the operating system. The first project by OpenBeOS was a community-created "stop-gap" update for BeOS 5.0.3 in 2002, featuring open source replacement for some BeOS components.[16] The kernel of NewOS, for x86, SuperH, and PowerPC architectures was successfully forked that same year,[17] and Haiku has been based on it since.[18] The app_server window manager was completed in 2005.[19] In July 2006, Haiku developer Stephan AÃŸmus introduced Icon-O-Matic, an icon editor, and a storage format (HVIF) with a rendering engine based on Anti-Grain Geometry.[20] The PackageInstaller was created by Åukasz Zemczak at the 2007 Google Summer of Code.[21] Java support was eventually added by a team from BeUnited who had ported it to BeOS,[22][23] followed by WLAN from the FreeBSD stack.[24] Alongside a port to GCC4, the first alpha release finally arrived after seven years of development.[25] Initially targeting full BeOS 5 compatibility, a community poll was launched to redefine the future of Haiku beyond a free software refactoring of BeOS from the late 1990s. It was decided to add support for contemporary systems, protocols, hardware, web standards, and compatibility with FLOSS libraries.[26] On October 27, 2009, Haiku obtained Qt4 support.[27] The WebPositive browser was first preloaded with Alpha2, replacing BeZillaBrowser.[28] After this, much time was spent on building a package management system, which went live in September 2013.[29] Beta1 arrived in 2018, and one of the most notable new features[30] was the PackageFS and package installation through the HaikuDepot and pkgman; Beta1 was the first official Haiku release to support full package management.[31] An initial port of Wine to Haiku began in 2022, allowing some Windows applications to run, though it remains under development as of 2025.[14]

Architecture

As with BeOS, Haiku is written in C++ and provides an object-oriented API.[39] The Haiku kernel is a modular hybrid kernel which began as a fork of NewOS, a modular microkernel written by former Be Inc. engineer Travis Geiselbrecht.[40] Many features have been implemented, including a virtual file system (VFS) layer and symmetric multiprocessing (SMP) support. It runs on 32-bit and 64-bit x86 processors, features an experimental port to RISC-V (Tier 2: bootloader boots with limited functionality as of October 2025), and has a port for ARM under development (also Tier 2, but further behind the x86 ports).[41][42][43] The application program interface (API) is based on that of BeOS, which is divided into a number of "kits" which collect related classes together and bear some relation to the library which contains the supporting code. In 2007, Access Co Ltd, the owners of Be, Inc's intellectual property, released the text of this (BeBook) under a Creative Commons licence.[44][45] The boot loader is filesystem agnostic and can also chainload GRUB, LILO and NTLDR. Since the Beta1 release, Haiku's memory management includes ASLR, DEP, and SMAP.[46] Graphics operations and window management is handled by the app_server protocol.[47] VESA is used as a fallback video output mode.[48] Haiku is POSIX compatible and has translation layers for X11 and Wayland.[49]

User interface

The graphical user interface is formed of Tracker, a file manager, and the Deskbar, an always-on-top taskbar that is placed in the upper right corner of the screen containing a menu, tray, and a list of running programs. Tracker is an evolution from OpenTracker, which was released under a license with two addenda restricting the use of Be Inc. trademarks.[50] Zeta also modified OpenTracker on their own operating system.[51] The icons in Haiku are named stippi and were designed by Stephan AÃŸmus.[52] AÃŸmus also created the Haiku Vector Icon Format (HVIF), a vector storage format to store icons in Haiku, and is aimed at fast rendering and small file sizes.[53] As of R1 Beta 5 (September 2024), Haiku supports a dark mode theme with light text on dark backgrounds and vice versa, along with simplified color selection offering three default options that auto-compute others, with manual editing available for advanced users.[54]

Software

Package management is done by the graphical application HaikuDepot, and a command-line equivalent called pkgman.[55] Packages can also be activated by installing them from remote repositories with pkgman, or dropping them over a special packages directory.[55] Haiku package management mounts activated packages over a read-only system directory. The Haiku package management system performs dependency solving with libsolv from the openSUSE project. As of October 2025, the official HaikuPorts repository provides over 4,300 packages via HaikuDepot.[56] It comes with a number of preloaded applications, such as a WebKit-based web browser WebPositive, a document reader BePDF, a simple web server PoorMan, text editors Pe and StyledEdit, an IRC client Vision, and a Bash-based terminal emulator Terminal.[57]

Compatibility with BeOS

Haiku R1 aims to be compatible with BeOS 5 at both the source and binary level,[58] allowing software written and compiled for BeOS to be compiled and run without modification on Haiku.[59] The 64-bit version of Haiku, however, does not have BeOS compatibility at the binary level, but the API still does. (The same would apply to other non-IA32 ports, such as RISC-V.)[59] Installation of these PKG format files is done using the PackageInstaller.[60]

Reception

In 2013 after the release of Haiku Alpha 4, Ars Technica reviewed the operating system and praised it for being fast, but ultimately stating that it "may not be much more than an interesting diversion, something to play with on a spare bit of hardware".[61] Haiku Beta 4 was reviewed by ZDNET in 2023 where it stated: "Haiku is for those who experienced either NeXT or AfterStep and want an operating system that looks and feels a bit old school but performs faster than any OS they've ever experienced." It further praised Haiku's kernel, file system, and object-oriented API.[62] The Free Software Foundation lists Haiku as a non-endorsed operating system because: "Haiku includes some software that you're not allowed to modify. It also includes nonfree firmware blobs."[63] Following the September 2024 release of Haiku R1 Beta 5, reception has been positive, with reviewers noting significant improvements in performance, UI polish, and hardware support. The Register described it as "sleek, slick, and fast to a degree no Linux distro can even rival," highlighting a 10x faster TCP/IP stack, enhanced Firefox support, and nearing daily-driver readiness as of January 2025.[64] Other outlets, such as How-To Geek and Hackaday, praised the simplified dark mode theme, USB audio support, and continued preservation of BeOS's minimalist philosophy.[65][66]
