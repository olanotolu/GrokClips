Cython

Cython (/ˈsaɪθɒn/) is a superset of the programming language Python, which allows developers to write Python code (with optional, C-inspired syntax extensions) that yields performance comparable to that of C.[5][6] Cython is a compiled language that is typically used to generate CPython extension modules. Annotated Python-like code is compiled to C and then automatically wrapped in interface code, producing extension modules that can be loaded and used by regular Python code using the import statement, but with significantly less computational overhead at run time. Cython also facilitates wrapping independent C or C++ code into Python-importable modules. Cython is written in Python and C and works on Windows, macOS, and Linux, producing C source files compatible with CPython 3.8 and later versions. The Cython source code that Cython compiles (to C) can use both Python 2 and Python 3 syntax, defaulting to Python 2 syntax in Cython 0.x and Python 3 syntax in Cython 3.x. The default can be overridden (e.g. in source code comment) to Python 3 (or 2) syntax. Since Python 3 syntax has changed in recent versions, Cython may not be up to date with the latest additions. Cython has "native support for most of the C++ language" and "compiles almost all existing Python code".[7]

Design

Cython works by producing a standard Python module. However, the behavior differs from standard Python in that the module code, originally written in Python, is translated into C. While the resulting code is fast, it makes many calls into the CPython interpreter and CPython standard libraries to perform actual work. Choosing this arrangement saved considerably on Cython's development time, but modules have a dependency on the Python interpreter and standard library. Cython has a foreign function interface for invoking C/C++ routines and the ability to declare the static type of subroutine parameters and results, local variables, and class attributes. A Cython program that implements the same algorithm as a corresponding Python program may consume fewer computing resources such as core memory and processing cycles due to differences between the CPython and Cython execution models. A basic Python program is loaded and executed by the CPython virtual machine, so both the runtime and the program itself consume computing resources. A Cython program is compiled to C code, which is further compiled to machine code, so the virtual machine is used only briefly when the program is loaded.[8][9][10][11][12] Cython employs:
  • Optimistic optimizations
  • Type inference (optional)
  • Low overhead in control structures
  • Low function call overhead[13][14]
Performance depends both on what C code is generated by Cython and how that code is compiled by the C compiler.[15] Cython is a derivative of the Pyrex language, but it supports more features and optimizations than Pyrex.[16][17] Cython was forked from Pyrex in 2007 by developers of the Sage computer algebra package, because they were unhappy with Pyrex's limitations and could not get patches accepted by Pyrex's maintainer Greg Ewing, who envisioned a much smaller scope for his tool than the Sage developers had in mind. They then forked Pyrex as SageX. When they found people were downloading Sage just to get SageX, and developers of other packages (including Stefan Behnel, who maintains the XML library LXML) were also maintaining forks of Pyrex, SageX was split off the Sage project and merged with cython-lxml to become Cython.[18] Following the initial release of Cython 0.1 on 28 July 2007, the project maintained a 0.x version series through numerous updates, reaching 0.29.37 in December 2023. This period focused on enhancing compatibility, performance, and Python 2/3 support. A major milestone came with Cython 3.0.0, released on 17 July 2023, which defaulted to Python 3 syntax (previously optional) and introduced improvements like better exception handling and C++ support. The 3.x series continued with 3.1.0 in May 2025 and the latest stable release, 3.1.6, on 23 October 2025.[19][3]

Hello World in Cython

A sample hello world program for Cython is more complex than in most languages because it interfaces with the Python C API and setuptools or other PEP 517-compliant extension building facilities. At least three files are required for a basic project using setup.py:
  • A setup.py file to invoke the setuptools build process that generates the extension module
  • A main Python program to load the extension module
  • Cython source file(s)

setup.py - build script for the extension module

python
from setuptools import setup
from Cython.Build import cythonize

setup(
    name="Hello world app",
    ext_modules=cythonize("*.pyx")

Alternative: Using pyximport for Development

For local development without a setup.py file, pyximport can automatically compile .pyx files on import. Install it with pip install pyximport, then use: 
python
import pyximport
pyximport.install()
import hello  # Compiles hello.pyx automatically if needed
hello.say_hello()
import pyximport
pyximport.install()
import hello  # Compiles hello.pyx automatically if needed
hello.say_hello()
This method is suitable for simple modules without extra C libraries or complex builds, but for distribution or advanced setups, use setup.py.[20]

Uses

Cython is particularly popular among scientific users of Python,[11][21][22] where it has "the perfect audience" according to Python creator Guido van Rossum.[23] Of particular note:
  • The free software SageMath computer algebra system depends on Cython, both for performance and to interface with other libraries.[24]
  • Significant parts of the scientific computing libraries SciPy, pandas and scikit-learn are written in Cython.[25][26][27]
  • Some high-traffic websites such as Quora used Cython (as of 2013).[28]
Cython's domain is not limited to just numerical computing. For example, the lxml XML toolkit is written mostly in Cython,[29] and like its predecessor Pyrex, Cython is used to provide Python bindings for many C and C++ libraries such as the messaging library ZeroMQ.[30] Cython can also be used to develop parallel programs for multi-core processor machines; this feature makes use of the OpenMP library.[31]
