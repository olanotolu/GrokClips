Apache CloudStack

CloudStack is open-source Infrastructure as a Service (IaaS) cloud computing software for creating, managing, and deploying IaaS cloud services. It uses existing hypervisor platforms for virtualization, such as KVM, VMware vSphere (including ESXi and vCenter), XenServer/XCP-ng. In addition to its own API, CloudStack also supports the Amazon Web Services (AWS) API.[1]

History

CloudStack was originally developed by Cloud.com, formerly known as VMOps.[2] VMOps was founded by Sheng Liang, Shannon Williams, Alex Huang, Will Chan, and Chiradeep Vittal in 2008.[3][4] The company raised a total of $17.6M in venture funding[5] from Redpoint Ventures, Nexus Ventures and Index Ventures (Redpoint and Nexus led the initial Series A funding round). The company changed its name from VMOps to Cloud.com on May 4, 2010, when it emerged from stealth mode by announcing its product.[6][2][7] Cloud.com was based in Cupertino, California. In May 2010, Cloud.com released most of CloudStack as free software under the GNU General Public License, version 3 (GPLv3).[8] They kept about 5% proprietary.[9] Cloud.com and Citrix both supported OpenStack, another Apache-licensed cloud computing program, at its announcement in July 2010.[10][11][12] In October 2010, Cloud.com announced a partnership with Microsoft to develop the code to provide integration and support of Windows Server 2008 R2 Hyper-V to the OpenStack project.[13] Citrix Systems purchased Cloud.com on July 12, 2011, for approximately $200 million.[14][15][16] In February 2012, Citrix released CloudStack 3.0. Among other features, this added support for Swift, OpenStack's S3-like object storage solution.[17] In April 2012, Citrix donated CloudStack to the Apache Software Foundation (ASF), relicensing the projectâ€”including the remaining proprietary codeâ€”under the Apache License version 2.0 and submitting it to the Apache Incubator. As part of this change, Citrix also ceased their involvement in OpenStack.[18] On November 6, 2012, CloudStack 4.0.0-incubating was announced,[19] the first stable release after joining ASF. On March 20, 2013, CloudStack graduated from Apache Incubator and became a Top-Level Project (TLP) of ASF.[20] The first stable (maintenance) release after graduation is CloudStack 4.0.2.[21] Since becoming a TLP in 2013, Apache CloudStack has been actively maintained and developed by the open-source community under the Apache Software Foundation. The project has produced numerous major and minor releases, with long-term support (LTS) versions introduced starting with 4.17.0.0 in June 2022 to provide extended stability and security updates. As of October 2025, the latest LTS release is 4.21.0.0, issued on August 28, 2025.[22]

Apache CloudStack Key Features

Key features

  • Rich user interface
  • noVNC-based VM console
  • Built-in high-availability for hosts and VMs
  • Hypervisor-agnostic
  • Multiple storage options including block and shared storage support
  • Snapshot management
  • Usage metering
  • Network management (VLAN, security groups)
  • Virtual routers, firewalls, load balancers
  • Multi-role support
  • LDAP, SAML, OAuth2, and 2FA support[23]
  • End-to-end encryption including secured console, volume encryption (since 4.18), and database encryption[24]
  • AWS API compatibility
  • Native autoscaling (since 4.18)[23]
  • IPv6 support for isolated and VPC networks (since 4.17)[23]
  • ARM64 support[23]

BareMetal hosts

BareMetal hosts in Apache CloudStack are physical servers managed without a hypervisor, typically via IPMI for provisioning. However, the BareMetal plugin is currently non-functional as of 2025.[38] Supported operating systems for BareMetal hosts (untested in recent releases):
  • Fedora 17
  • Ubuntu 12.04[39]
Alternatives such as integration with Canonical's MAAS via the External Deployment feature are being explored.[40]

Deployment architecture

The minimum production installation consists of one machine running the CloudStack Management Server and another machine to act as the cloud infrastructure (in this case, a very simple infrastructure consisting of one host running hypervisor software). In its smallest deployment, a single machine can act as both the Management Server and the hypervisor host (using the KVM hypervisor).[41] Multiple management servers can be configured for redundancy and load balancing, all pointing to a common MySQL database.

Users

Apache CloudStack is used by over 150 organizations worldwide, including service providers, product vendors, private cloud operators, and systems integrators.[42] Notable service providers include Angani, Bell Canada, China Telecom, Leaseweb, and Verizon. Product vendors such as Citrix Systems, Dell, Huawei, Juniper Networks, and SAP integrate CloudStack into their offerings. Private cloud users encompass companies like Apple, Disney, Globo.com, and the University of Melbourne. Systems integrators include ShapeBlue, Accudata Systems, and Business Connexion.[42] In July 2012, Datapipe launched what was then reported as the largest international public cloud built on CloudStack, spanning six data centers in the United States (California, Virginia, New York), Britain (London), and Asia (Shanghai, Hong Kong).[43]
